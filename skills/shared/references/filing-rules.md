# Filing Rules

Use the smallest durable filing decision that keeps later retrieval reliable.

- Append raw notes to a thread when they are part of one ongoing session.
- Create or update a topic only for a durable subject likely to matter again.
- Keep one canonical action-item record per task or unresolved question.
- Use links and references instead of duplicating the same content into multiple files.

Prefer these retrieval surfaces:

1. `memory/README.md`
2. generated views
3. typed topic indexes
4. topic/action-item records
5. thread snippets or normalized import text

If information is ambiguous, preserve the ambiguity instead of forcing a clean filing decision too early.
