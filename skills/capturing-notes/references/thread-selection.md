# Thread Selection

Use this order:

1. If there is exactly one clearly related open thread, append to it.
2. If there is no clearly related open thread, start a new thread.
3. If multiple open threads are plausible, ask the user.
4. If the best thread last captured a note on a prior day, treat it as stale-open and ask whether to resume or start a new thread.

Closed threads stay closed unless:

- the user explicitly asks to reopen one, or
- you have strong evidence it is the same bounded work item and you ask first.
