# Capture Edge Cases

Ask the user when:

- more than one open thread looks plausible
- a stale-open thread might or might not be the right place to continue
- the note likely belongs to a closed thread
- the note implies important action state but the intent is too ambiguous

Prefer a new thread when:

- the note starts a clearly different workstream
- the prior open thread is stale and not clearly related

Prefer appending when:

- there is one clearly related current thread
- the user is obviously continuing the same session

Never block raw capture on perfect categorization. Capture first, then clarify.
